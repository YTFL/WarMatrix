## How to contribute to WarMatrix

#### **Did you find a bug?**

* **Do not open up a GitHub issue if the bug is a security vulnerability**, and instead refer to our security policy or contact the maintainers privately.

* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/WarMatrixAI/WarMatrix/issues).

* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/WarMatrixAI/WarMatrix/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a code sample or steps to reproduce the issue.

#### **Did you write a patch that fixes a bug?**

* Open a new GitHub pull request with the patch.

* Ensure you fill out the [🐛 Bug Fix Pull Request Template](https://github.com/WarMatrixAI/WarMatrix/tree/main/.github/PULL_REQUEST_TEMPLATE/bug_fix.md) completely. This includes linking the relevant issue number (`Fixes #...`) and checking the **"Allow edits from maintainers"** box so we can collaborate with you easily.

#### **Do you intend to add a new feature or change an existing one?**

* Suggest your change in the [GitHub Discussions Ideas](https://github.com/orgs/WarMatrixAI/discussions/categories/ideas) before writing extensive code.

* Do not open an issue on GitHub until you have collected positive feedback about the change. GitHub issues are primarily intended for bug reports and fixes. When you do submit a PR for a feature, make sure to select the [✨ New Feature Template](https://github.com/WarMatrixAI/WarMatrix/tree/main/.github/PULL_REQUEST_TEMPLATE/feature.md).

#### **Are you introducing a breaking change?**

* If your pull request alters existing behavior or requires other developers to change their setup, please use our specific [💥 Breaking Change Template](https://github.com/WarMatrixAI/WarMatrix/tree/main/.github/PULL_REQUEST_TEMPLATE/breaking_change.md) to explain the migration impacts.

#### **Do you have questions about the source code?**

* Ask any question about how to use or build upon WarMatrix in [GitHub Discussions Q&A](https://github.com/orgs/WarMatrixAI/discussions/categories/q-a).

#### **Do you want to contribute to the WarMatrix documentation?**

* Documentation updates are highly appreciated! Submit a Pull Request and select our [📝 Documentation Update Template](https://github.com/WarMatrixAI/WarMatrix/tree/main/.github/PULL_REQUEST_TEMPLATE/documentation.md). Make sure your markdown formatting renders correctly.

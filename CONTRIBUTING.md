## How to contribute to WarMatrix

#### **Did you find a bug?**

* **Do not open up a GitHub issue if the bug is a security vulnerability in Rails**, and instead refer to our [security policy]().

* **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/WarMatrixAI/WarMatrix/issues).

* If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/WarMatrixAI/WarMatrix/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or an **executable test case** demonstrating the expected behavior that is not occurring.

#### **Did you write a patch that fixes a bug?**

* Open a new GitHub pull request with the patch.

* Ensure the PR description clearly describes the problem and solution. Include the relevant issue number if applicable.

#### **Do you intend to add a new feature or change an existing one?**

* Suggest your change in the [Github Discussions Ideas](https://github.com/orgs/WarMatrixAI/discussions/categories/ideas) and start writing code.

* Do not open an issue on GitHub until you have collected positive feedback about the change. GitHub issues are primarily intended for bug reports and fixes.

#### **Do you have questions about the source code?**

* Ask any question about how to use WarMatrix in [Github Discussions Q&A](https://github.com/orgs/WarMatrixAI/discussions/categories/q-a).

#### **Do you want to contribute to the Rails documentation?**

* Please request approval by submitting a pull Request.*

* WarMatrix is a volunteer effort. We encourage you to pitch in and join the team! Please submit a request in [Github Discussions General](https://github.com/orgs/WarMatrixAI/discussions/categories/general)*
